## 主要结果
在本实验中，我们进行了以下实验并得到结果：

1. **缓存与剪枝算法优化效率测试**：
   我们测试了缓存机制以及alpha-beta剪枝算法对效率的优化效果。实验结果表明，单独使用剪枝算法以及混合使用剪枝和缓存两种方法均比只使用缓存方法要慢。混合使用两种方法时，容易导致缓存结果的误用。

   混合策略容易出错以及性能下降的主要原因是缓存存储的结果不再是节点的精确值，而是变成了一个区间范围内的值，这导致下一次访问相同状态时，其对应的区间可能发生变化，从而无法命中原来的缓存。

2. **动态规划与多进程加速**：
   在同学的指导下，我们采用了动态规划（DP）和多进程技术来处理后续计算任务，获得了显著的加速效果。动态规划加速的主要原因是避免了Python创建函数调用栈的开销。

3. **Cython优化尝试**：
   我们尝试使用Cython进行性能优化，但由于lru_cache机制仍保留在Python层面，导致优化效果有限。

4. **CPU算力需求分析**：
   本实验要求CPU算力充分。多进程技术需要CPU具备足够的计算能力，如果有其他进程占用了算力，则会显著影响本实验的运行效率。

5. **状态哈希预处理**：
   我们对游戏状态进行了哈希预处理。虽然在运行效率上未观察到显著变化，但这一处理在动态规划算法的实现中提供了良好的状态表示方式，提高了编码效率。
